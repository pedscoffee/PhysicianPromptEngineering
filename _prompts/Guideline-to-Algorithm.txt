Convert the provided text guideline into a step-by-step clinical algorithm.

Input: Dense Text Guideline.
Output: Logical Algorithm/Flowchart text.

Output Format:
**ALGORITHM: [Topic]**

**Step 1: [Entry Criteria]**
- If [Condition A] -> Go to Step 2.
- If [Condition B] -> Stop. [Action].

**Step 2: [Decision Point]**
- **Is [Variable] Present?**
    - **YES**: [Action] -> Go to Step 3.
    - **NO**: [Action] -> Go to Step 4.

**Step 3: [Treatment/Action]**
- [Specific Dose/Duration]

**Step 4: [Alternative]**
- [Specific Dose/Duration]

Rules:
1.  **Linear Logic**: Must flow logically from start to finish.
2.  **Binary Decisions**: Break complex decisions into Yes/No questions.
3.  **Action Oriented**: Every endpoint must be an action (Treat, Test, or Stop).

Few-Shot Examples:
Input:
"For patients with CAP, assess CURB-65. If 0-1, treat outpatient with Amox or Doxy. If 2, admit. If >3, consider ICU. If ICU, use Ceftriaxone + Azithro."

Output:
**ALGORITHM: Community Acquired Pneumonia Triage**

**Step 1: Calculate CURB-65 Score**
- Confusion, Urea >7, RR >30, BP <90/60, Age >65.

**Step 2: Triage Decision**
- **Score 0-1?**
    - **Action**: Treat Outpatient.
    - **Meds**: Amoxicillin OR Doxycycline.
- **Score 2?**
    - **Action**: Admit to Ward.
- **Score 3-5?**
    - **Action**: Admit (Consider ICU).

**Step 3: Inpatient Treatment (if Admitted)**
- **Ward**: Ceftriaxone + Azithromycin.
- **ICU**: Ceftriaxone + Azithromycin (consider adding MRSA coverage if risk factors).

Quality Checks:
1. Is the flow logical?
2. Are the endpoints clear?
