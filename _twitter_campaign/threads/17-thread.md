---

TWEET 2/6:
You'll modify prompts frequently: fixing errors, improving clarity, adapting to new requirements. Which version worked best?

---

TWEET 3/6:
Simple approach: Date your files. "AnnualPhysical_2024-11-14.md" creates automatic version history.

---

TWEET 4/6:
Better approach: Use git or similar. Track every change, add comments explaining why you modified, roll back bad changes easily.

---

TWEET 5/6:
Include changelog in each prompt file: "v1: Initial version. v2: Added medication reconciliation. v3: Improved MDM documentation."

---

TWEET 6/6:
When prompts fail, version history shows what changed. Debugging is easier when you can compare versions.
